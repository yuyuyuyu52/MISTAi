# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

Single-page marketing site for **MIST Ai** (AI 产业化全周期增长与资本服务平台). Bootstrapped as a Google AI Studio app — copy is bilingual (EN headings, ZH body). Currently there is no real Gemini usage despite `@google/genai` being installed; the dependency and `GEMINI_API_KEY` plumbing exist because AI Studio auto-injects them.

## Commands

```bash
npm run dev      # vite dev server on 0.0.0.0:3000
npm run build    # vite build → dist/
npm run preview  # serve dist/
npm run lint     # tsc --noEmit  (this repo's only typecheck/lint step)
npm run clean    # rm -rf dist
```

There is no test suite and no ESLint. `npm run lint` is literally just `tsc --noEmit` — run it before declaring work done.

## Environment

Copy `.env.example` to `.env.local`. `GEMINI_API_KEY` is exposed to client code as `process.env.GEMINI_API_KEY` via Vite's `define` (see `vite.config.ts`) — it is **not** the usual `import.meta.env.VITE_*` pattern. Setting `DISABLE_HMR=true` turns HMR off (AI Studio does this during agent edits to prevent flicker).

## Architecture

**Stack:** Vite 6 + React 19 + TypeScript (strict-ish, `noEmit`), Tailwind CSS v4, Motion (ex–framer-motion), Base UI, shadcn-style primitives, lucide-react icons. `express` + `tsx` are installed but no server exists yet — likely stubs for a future AI Studio/Cloud Run adapter.

**Entry flow:** `index.html` → `src/main.tsx` → `src/App.tsx`. `App.tsx` is the whole site: every section (`Nav`, `Hero`, `Services`, `Portfolio`, `Global`, `Footer`, plus `ServiceCard`/`PortfolioItem`/`FloatingOrb`) is an inline component in that one file. If you extract sections into separate files, follow the shadcn alias convention below.

**Path alias:** `@/*` resolves to the **project root**, not `src/`. So imports look like `@/components/ui/button`, `@/lib/utils`. This is mirrored in both `tsconfig.json` and `vite.config.ts` — update both if you change it. shadcn's `components.json` encodes these aliases (`components: @/components`, `ui: @/components/ui`, `lib: @/lib`, `utils: @/lib/utils`, `hooks: @/hooks`). Adding a component via `npx shadcn add` will drop files into `/components/ui` — not under `src/`.

**Tailwind v4:** Config is CSS-first in `src/index.css` using `@import "tailwindcss"` and a `@theme` block — there is no `tailwind.config.js`. Brand tokens are defined there as `--color-mist-green` (`#00d98b`), `--color-mist-blue` (`#0070f3`), `--color-mist-accent` (`#7928ca`), and surface/border rgba values. They are available as utilities (`bg-mist-green`, `text-mist-blue`, `border-mist-border`, etc.). Fonts: `--font-sans` = Inter, `--font-display` = Outfit (both loaded from Google Fonts in the CSS).

**Design system conventions used throughout `App.tsx`:**
- Custom utility classes defined in `index.css`: `heading-xl`, `heading-lg`, `glass-card`, `cool-gradient-text`, `fresh-bg`, `floating-orb`, `btn-cool`, `noise-overlay`. Reuse these instead of re-deriving the same clamp/gradient/backdrop-blur combos inline.
- Heavy use of Motion's `motion.div` + `whileInView` for scroll-triggered reveals and continuous ambient animations (rotating rings, floating orbs). When adding sections, match this aesthetic — the site leans into animated gradients, blurred orbs, and gloss glassmorphism.
- Uppercase-tracked labels (`tracking-[0.2em] uppercase text-[10px] font-bold`) are a recurring motif for eyebrow/CTA text.

## Notes on this codebase

- The repo was generated by AI Studio (`ai.studio/apps/70ae3b07-...`); `package.json` name is still the template default `react-example`.
- The noise-overlay SVG string in `src/index.css` has malformed entities (`%3%3F`) — it renders as no-op. Don't "fix" it unless the user asks.
- `FloatingOrb`, `ServiceCard`, and `PortfolioItem` use `any` props. If you touch these, consider typing them, but don't refactor preemptively — the user may prefer the file stays compact.
